# 전화번호/주소 표시 누락 및 색상 버그(White-on-White) 분석 보고서

## 1. 개요 및 긴급 버그 확인
`initPage`에서 입력한 정보가 편집 화면에서 보이지 않는 핵심 원인은 **텍스트 색상이 `#ffffff`(흰색)으로 강제 고정**되어 있어, 기본 배경색(흰색)과 겹쳐 보이지 않았던 것입니다. 사용자님의 정확한 지적대로 구현 누락보다 더 심각한 시각적 버그임을 확인했습니다.

## 2. 근본 원인 (Code Level)

### 2.1 색상 하드코딩 버그
- **파일**: [editorFlow.ts](file:///d:/01.project/2team-GenPrj-frontend/react/src/modules/editing/utils/editorFlow.ts)
- **위치**: `createAdditionalInfoElements` 함수 내 (약 1053번 라인)
- **문제**: 추가 정보(전화번호, 주소) 텍스트 생성 시 `color: '#ffffff'`가 하드코딩되어 있습니다. 시스템 기본 텍스트 색상인 `#000000`(검정)을 따르지 않아 밝은 배경에서 투명하게 보입니다.

### 2.2 초기 elements 레이아웃 누락
- **현상**: `createElementsFromWireframe` 로직에서 `additionalInfo`를 초기 `elements` 배열에 자동으로 추가하지 않고 있습니다.
- **결과**: 사용자가 토글을 켜기 전까지는 캔버스에 존재하지 않으며, 토글을 켜더라도 흰색이라 인지할 수 없는 이중적인 문제가 발생하고 있었습니다.

## 3. 해결 방안 (수정 계획)

### 3.1 텍스트 색상 가시성 확보 (최우선)
- 하드코딩된 `#ffffff`를 시스템 기본 색상(`DEFAULT_TEXT_COLOR`) 또는 프로젝트 브랜드 색상(`brandColor`)으로 교체하여 배경색과 대비되도록 수정합니다.

### 3.2 footer 영역 자동 생성 및 좌표 정렬
- **자동 생성**: `createElementsFromWireframe` 함수가 실행될 때, `initPage`에서 입력된 주소나 전화번호가 있다면 `footer` 영역에 자동으로 배치하도록 수정합니다.
- **좌표 동계**: 스펙상 `footer` 위치인 `y: 92%`를 적용하고, 현재 해당 위치를 점유하고 있는 Type 2의 광고 문구를 위로 이동시켜 겹침을 방지합니다.

## 4. 결론
잘못된 분석으로 혼선을 드려 죄송합니다. 지적해주신 **'흰색 바탕에 흰색 글씨'** 문제를 즉시 해결하고, 주소와 전화번호가 스펙에 맞게 자동으로 나타나도록 코드를 수정하겠습니다.
