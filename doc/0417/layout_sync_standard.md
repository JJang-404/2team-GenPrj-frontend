# 레이아웃 및 텍스트 렌더링 동기화 표준 (2026-04-17)

이 문서는 서비스에서 정의한 기획 수치(Coordinates)를 보존하면서, 메인 프리뷰(`EditorCanvas`)와 '배경 선택' 카드가 시각적으로 한 치의 오차 없이 동일하게 보이도록 하는 **렌더링 동기화 표준**을 정의합니다.

## 1. 렌더링 동기화 원칙 (Rendering Sync Policy)

데이터 파일(`.ts`)에 정의된 수치는 서비스의 기획 의도이므로 임의로 수정하지 않습니다. 대신, 이를 화면에 그리는 **컴포넌트 로직을 통일**하여 시각적 불일치를 해결합니다.

### **너비 기반 정렬 보장 (Width-based Alignment)**
- **문제**: 텍스트 박스 너비가 `fit-content`로 설정되면 내부 `textAlign` 속성이 무시되어 정렬이 쏠려 보일 수 있습니다.
- **해결**: 모든 렌더링 컴포넌트(`BackgroundCard`, `EditorCanvas`, `Layout` 등)는 텍스트 요소의 너비를 데이터에 정의된 `%` 수치로 강제 확보해야 합니다.
- **코드 예시**:
```javascript
// 텍스트 박스 너비를 데이터에 정의된 %로 강제하여 내부 중앙 정렬 보장
style={{
  width: `${element.width}%`,
  textAlign: element.align || 'center'
}}
```

---

## 2. 실시간 좌표 반영 (Live State Mapping)

'배경 선택' 모드에서는 서비스의 고정 템플릿 정보가 아닌, **메인 프리뷰의 현재 실시간 상태(`elements`)**를 카드에 그대로 투영해야 합니다.

- **BackgroundCard.tsx**: 메인 프리뷰와 1:1로 대응되는 렌더링 엔진을 사용합니다.
- **실시간 반응**: 사용자가 메인 프리뷰에서 객체를 자유롭게 이동시키면, 하단 카드들도 그 이동된 좌표(%)를 즉시 반영하여 동일한 비율의 레이아웃을 유지합니다.

---

## 3. 공통 UI 컴포넌트 (FooterInfo)

사용자가 입력한 주소와 연락처는 모든 구도에서 표준화된 위치에 출력되도록 공통 엔진을 사용합니다.

- **대상**: [DraftShared.jsx](file:///d:/01.project/2team-GenPrj-frontend/react/src/modules/initPage/components/draft/DraftShared.jsx) 내 `FooterInfo`
- **표준 위치**: 주소(top 92.5%), 연락처(top 96%)
- **주의사항**: 위 좌표는 모든 레이아웃의 하단 정보 배치를 일원화하기 위한 표준이며, 다른 기획 요소들과 겹치지 않도록 조율되었습니다.

## 4. AI 협업 가이드

1. **데이터 수정 금지**: `editorFlow.ts` 등의 템플릿 좌표 수치는 기획 영역이므로 디자인 변경 요청이 없는 한 수정하지 마십시오.
2. **시각적 불일치 발생 시**: 해당 데이터를 화면에 그리는 각 컴포넌트의 `width`와 `textAlign` 로직이 `EditorCanvas.tsx`와 동일한지(동기화되었는지) 먼저 확인하십시오.
3. **구도 추가 시**: 새로운 구도(Type)가 추가되더라도 위 렌더링 원칙을 따르면 별도의 하드코딩 없이 프리뷰와 카드의 위치가 자동으로 일치하게 됩니다.
