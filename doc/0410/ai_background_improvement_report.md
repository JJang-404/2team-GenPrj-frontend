# AI 배경 생성 품질 개선 결과 보고서 (2026.04.10)

사용자님이 제기하신 **"배경 생성 시 객체 반복 현상"**과 **"프롬프트 무시 문제"**를 해결하기 위해 프론트엔드 로직을 개선하였습니다.

## 1. 주요 개선 사항

### ① 업종(Industry) 및 브랜드 컨텍스트 자동 주입
- **기존**: 사용자의 입력값만 전달 (예: "봄 분위기로")
- **개선**: `storeInfo`에 저장된 **가게 이름**과 **업종** 정보를 자동으로 추출하여 프롬프트에 결합합니다.
  - *예시*: 업종이 '카페'인 경우, `setting: cafe interior/exterior` 키워드가 자동으로 추가되어 커피잔 뒤에 자연스러운 카페 배경이 그려지도록 유도합니다.

### ② 생성 강도(Strength) 최적화
- **기존**: `0.3` 또는 `0.72` (배경을 바꾸기에 다소 약한 값)
- **개선**: 모든 배경 교체 작업의 기본 강도를 **`0.85`**로 상향 조정했습니다. 
  - 이를 통해 AI가 기존의 흰색 배경에 얽매이지 않고 프롬프트에 충실한 새로운 배경을 그리게 됩니다.

### ③ 프롬프트 구조 고도화
- **기술적 지시 최소화**: AI가 혼란을 느끼지 않도록 기술적인 지시사항을 간결하게 정리하고, 시각적 묘사(Visual Description)를 최상단에 배치했습니다.
- **네거티브 프롬프트 강화**: 배경에 동일 객체가 생기지 않도록 `duplicate subject in background` 키워드를 추가했습니다.

## 3. 확인 방법 (테스트 가이드)

1.  **업종 설정**: 설정 메뉴에서 업종을 '카페' 또는 '식당'으로 설정되어 있는지 확인합니다.
2.  **배경 생성**: "봄이랑 어울리는 배경으로"라고 입력하고 버튼을 누릅니다.
3.  **결과 확인**:
    - 배경에 업종과 어울리는 풍경(카페 내부 등)이 나타나는지 확인합니다.
    - 커피잔 옆에 또 다른 커피잔이 생기는 '반복 현상'이 줄어들었는지 확인합니다.
