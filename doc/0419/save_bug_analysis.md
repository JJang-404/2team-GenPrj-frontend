# 배경색 저장 버그 수정 및 분석 보고서 (0419)

'전체 저장' 기능 사용 시 단색 외에 그라데이션 및 다중색(Pastel) 배경이 정상적으로 저장되지 않던 문제를 분석하고 해결한 내용입니다.

## 1. 문제 현상
- '전체 저장' 버튼 클릭 시, 배경이 단색인 경우에만 정상 저장됨.
- 그라데이션 또는 다중색 배경인 경우, 첫 번째 단색으로만 저장되거나 배경색이 누락됨.

## 2. 원인 분석
### 2.1 html2canvas의 배경 렌더링 한계
- 현재 `EditorCanvas`는 `.editor-stage__canvas` 컨테이너 내부에 배경 전용 div(`.editor-stage__background`)를 두고 그라데이션을 적용합니다.
- 포스터 캡처에 사용되는 `html2canvas` 라이브러리는 중첩된 요소의 복잡한 CSS 그라데이션(`linear-gradient`)을 파싱하는 과정에서 오류가 발생하거나, 첫 번째 색상 토큰만 인식하여 렌더링하는 고질적인 문제가 있습니다.

### 2.2 captureMode에서의 배경 렌더링 누락
- `EditorCanvas.tsx` 코드 내에 `!captureMode` 조건문이 있어, 캡처 시점에 배경 요소 자체가 DOM에서 생략되는 로직이 포함되어 있었습니다.

## 3. 해결 방안 (적용 완료)

### 3.1 캔버스 컨테이너 직접 배경 적용 (EditorCanvas.tsx)
- `.editor-stage__canvas` (부모 컨테이너)에 직접 `background` 스타일을 적용합니다.
- 기존의 별도 배경 div 레이어를 제거하고 부모 엘리먼트에 직접 스타일을 부여함으로써, `html2canvas`가 레이어 레이아웃을 계산하는 과정에서 발생할 수 있는 오차와 렌더링 누락을 원천 차단하였습니다.

### 3.2 표준 CSS 키워드 및 명시적 구간 선언 (Shared & App.tsx)
- `90deg`, `180deg` 대신 브라우저 호환성이 더 높은 `to right`, `to bottom` 키워드를 사용합니다.
- `0%`와 `100%` 시작/종료 지점을 명시하여 `html2canvas`가 그라데이션 구간을 오인하지 않도록 최적화하였습니다.
- `App.tsx`의 `handleFullSave`에서 수동으로 스타일을 주입하던 로직을 제거하고, 컴포넌트 자체 렌더링에 의존하도록 하여 상태 불일치 가능성을 제거했습니다.

### 3.3 EditorCanvas 렌더링 로직 수정 (EditorCanvas.tsx)
- `captureMode`일 때도 배경 div와 생성 이미지가 렌더링되도록 조건문을 제거하였습니다.
- `backgroundColorDraft`와 `backgroundMode`를 props로 받아 상태 동기화를 더욱 견고하게 하였습니다.

## 4. 추가 확인 사항: `useProducts.js` 영향도
- `react/src/modules/initPage/hooks/useProducts.js`는 `initPage`에서 제품 정보와 제품 이미지의 배경 제거를 관리하는 Hook입니다.
- 확인 결과, 해당 파일은 제품 리스트 상태만 관리하며 **편집 페이지(`editing`)의 전체 배경색 상태나 저장 모듈에는 직접적인 영향을 주지 않습니다.**
- 따라서 이번 버그는 제품 관리 로직보다는 **화면 캡처 엔진(html2canvas)과 스타일 렌더링 구조 간의 부조화**에서 기인한 것으로 판단됩니다.
