# [Plan] 메인 프리뷰와 선택 카드 간 텍스트 비율 동기화 (v3)

작성일: 2026-04-15
목표: 'Main Preview'와 'WireframeChoiceCard'의 텍스트 크기가 각각의 컨테이너 너비에 비례하도록 수정하여, 시각적인 "밀도"가 동일하게 보이도록 개선합니다.

## 1. 현재 문제점 분석

1. **절대 폰트 크기 사용**: 현재 'Main Preview'는 캔버스의 실제 화면 너비와 관계없이 요소에 설정된 `px` 크기(예: 36px) 그대로 텍스트를 렌더링합니다.
2. **비율적 불일치**: 캔버스 너비가 580px일 때 36px과, 카드의 너비가 310px일 때의 36px은 시각적 비율이 완전히 다릅니다. 사용자의 눈에는 두 텍스트의 절대적인 크기가 비슷하게 보여서 비율이 깨진 것으로 느껴집니다.
3. **스케일링 미적용**: 'Main Preview'에 `ResizeObserver`를 통한 폰트 크기 보정 로직이 누락되어 있어, 화면 크기가 변해도 텍스트가 작아지지 않습니다.

## 2. 해결 방안 (v3)

### 2-1. 공통 기준 너비(Reference Width) 설정
- 모든 폰트 크기의 기준이 되는 너비를 `580px`로 통일합니다 (가이드 문서 기준).

### 2-2. Main Preview (EditorCanvas) 보정
- `EditorCanvas.tsx`에 `ResizeObserver`를 추가하여 현재 캔버스의 실제 너비(`W`)를 실시간으로 측정합니다.
- `scaleFactor = W / 580`을 계산합니다.
- 모든 텍스트 요소의 `fontSize`와 `letterSpacing`에 이 `scaleFactor`를 곱하여 적용합니다.
    - 예: 기준 36px, 현재 너비 290px(50%) -> 실제 렌더링 폰트 18px.

### 2-3. 선택 카드 (BackgroundCard & WireframeChoiceCard) 보정
- `BackgroundCard.tsx`와 `WireframeChoiceCard.tsx` 모두 메인 프리뷰의 편집 상태를 실시간으로 반영합니다.
- 특히 `BackgroundCard`는 `px` 단위의 `fontSize`를 그대로 노출하여 텍스트가 거대하게 보이는 문제가 있습니다.
- 모든 선택 카드에 `ResizeObserver`를 통한 폰트 크기 비율 동기화 로직을 적용합니다.

### 2-4. '최종 광고 문구' 중복 노출 해결
- 현재 메인 프리뷰에서 광고 문구가 2개로 나오는 원인을 분석하고, 템플릿 필드와 폴백 필드 중 하나만 노출되도록 필터링 로직을 강화합니다.
- `mapProjectDataToTemplate` 로직에서 중복 매칭을 방지하고, 하나의 핵심 문구만 유지합니다.

## 3. 작업 계획

| 단계 | 파일 | 작업 내용 |
|---|---|---|
| **1단계** | `EditorCanvas.tsx` | `ResizeObserver` 추가 및 `px` 단위 폰트 크기에 `scaleFactor` 반영 |
| **2단계** | `WireframeChoiceCard.tsx` | `scaleFactor` 적용 로직 점검 및 메인 프리뷰와의 비율 교차 검증 |
| **3단계** | `index.css` / 스타일 | 컨테이너 사이즈 변화에 따른 자동 반응형 동작 확인 |

## 4. 기대 효과
- 사이드바를 접거나 펼칠 때, 혹은 창 크기를 조절할 때 메인 프리뷰의 텍스트가 캔버스 크기에 맞춰 자연스럽게 커지거나 작아집니다.
- 우측의 선택 카드들은 메인 프리뷰의 "미니어처" 버전처럼 완벽하게 동일한 비율의 텍스트 크기를 유지하게 됩니다.
