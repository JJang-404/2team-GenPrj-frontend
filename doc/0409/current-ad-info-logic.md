# 에디팅 페이지 광고 정보 및 AI 문구 기능 분석

현재 `editingPage`의 사이드바 내 **'광고 정보'** 섹션과 **'AI 문구'** 버튼의 내부 동작 방식에 대한 분석 결과입니다.

## 1. 광고 정보 (Ad Info) 섹션의 역할
이 섹션은 `initPage`에서 넘어온 핵심 텍스트 정보를 확인하고 직접 수정할 수 있는 UI입니다.
- **가게 이름 (Store Name)**: 매장 명칭을 표시하며, 수정 시 캔버스의 모든 '가게명' 요소가 동기화되어 변경됩니다.
- **메인 슬로건 (Main Slogan)**: 광고의 핵심 문구를 표시합니다. 사용자가 직접 입력하거나 'AI 문구' 버튼을 통해 자동 생성된 내용을 확인할 수 있습니다.

## 2. AI 문구 (Auto Slogan) 생성 로직
현재 시스템의 'AI 문구' 버튼은 실제 외부 AI(GPT 등)를 실시간으로 호출하지 않고, **클라이언트 측 유틸리티(`projectEditor.ts`)에 정의된 규칙**에 따라 동작합니다.

### 2-1. 동작 원리 (`createAutoSlogan` 함수)
1. **데이터 추출**: 현재 설정된 `가게 이름`과 상품 목록 중 첫 번째 `상품 이름`을 가져옵니다.
2. **폴백(Fallback) 처리**: 정보가 없을 경우 '우리 가게', '시그니처 메뉴'라는 기본 단어를 사용합니다.
3. **템플릿 조합**: 미리 정의된 4가지 문구 패턴 중 하나를 무작위(Random)로 선택하여 완성합니다.
   - *예시 패턴*: `{가게 이름}의 {상품 이름}, 지금 가장 선명한 한 잔`

### 2-2. 캔버스 반영 과정
- 사용자가 버튼을 누르면 위 로직으로 생성된 문자열이 `projectData.mainSlogan`에 저장됩니다.
- 동시에 `applyProjectTextField` 함수가 실행되어, 캔버스 위에 배치된 모든 텍스트 요소 중 `메인 타이틀` 또는 `광고 문구` 라벨을 가진 요소들의 텍스트를 즉시 교체합니다.

## 3. 추가 정보 (Additional Info) 토글 기능
사이드바 하단의 '좌석수', '주소', '노키즈존' 등의 버튼은 **동적 레이어 추가/삭제** 방식으로 동작합니다.
- 버튼을 켜면(`True`): 해당 정보에 맞는 아이콘 이미지와 텍스트 요소가 캔버스의 특정 좌표에 생성됩니다.
- 버튼을 끄면(`False`): 해당 라벨을 가진 요소들이 캔버스에서 제거됩니다.

## 4. 향후 개선 방향 (GPT 연동 시)
현재는 단순한 문자열 조합 방식이므로, 향후 GPT 연동 시 다음과 같은 변화가 일어납니다.
- **기존**: 고정된 4개 문구 중 랜덤 선택
- **변경**: `Backend_final`을 통해 가게 업종, 주소, 상품 특징을 GPT에 전달하여 매번 새로운 맥락의 문구를 생성 (예: "2층 엘리베이터가 있어 유모차도 편한 {가게이름}에서 {상품이름}을 즐기세요")
