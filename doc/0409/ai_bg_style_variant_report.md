# AI 배경 생성 스타일 다양화 적용 보고서

## 1. 개요
기존에 동일한 프롬프트로 4개의 배경을 생성하던 방식에서 벗어나, 사용자가 입력한 프롬프트를 유지하면서 각기 다른 4가지 스타일을 적용하도록 개선했습니다.

## 2. 주요 변경 사항

### 2.1 4가지 스타일 변동성 추가 (`modules/editing/App.tsx`)
- 배경 생성 시 다음 4가지 키워드를 자동으로 조합하여 요청합니다:
  1. **분위기 고급지게**
  2. **빈티지**
  3. **세련되게**
  4. **카툰화**
- **조합 방식**: 사용자가 입력한 프롬프트(`promptKo`) 뒤에 해당 스타일 키워드를 콤마(,)로 연결하여 `customPrompt`로 전달합니다. (예: "커피숍 내부, 빈티지")

### 2.2 UI 시각적 피드백 개선
- 생성된 각 배경 후보군(Background Candidates)의 이름에 적용된 스타일을 명시했습니다.
  - 예: `AI 배경 1 (분위기 고급지게)`, `AI 배경 2 (빈티지)` 등
- 카드 설명(Note)란에도 해당 스타일이 적용되었음을 표시하여 사용자가 선택 시 혼동하지 않도록 했습니다.

### 2.3 프롬프트 강제성 완화 및 연동
- `bootstrap.ts`에서 각 템플릿별로 정의된 `defaultPromptKo`가 `promptKo` 상태의 초기값으로만 사용되도록 구조를 명확히 했습니다.
- 사용자가 사이드바에서 프롬프트를 수정하면, 수정된 내용이 즉시 4가지 스타일 변동성과 결합되어 서버로 전송됩니다.

## 3. 작업 결과
- 이제 한 번의 클릭으로 4가지 서로 다른 감성의 배경 결과물을 제안받을 수 있습니다.
- 브라우저 콘솔로그를 통해 각 요청(`배경 요청 #1~4`)에 어떤 스타일이 포함되었는지 투명하게 확인할 수 있습니다.

## 4. 향후 권장 사항
- 특정 스타일에 대한 사용자 선호도를 분석하여 기본 스타일 후보군을 교체하거나 확장할 수 있습니다.
- 스타일 키워드에 따른 번역 품질 향상을 위해 백엔드 `openai.py`의 가이드를 보완할 수 있습니다.
