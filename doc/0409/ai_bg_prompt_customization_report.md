# AI 배경 생성 프롬프트 독립화 및 사용자 정의 개선 보고서

## 1. 개요
기존에 `bootstrap.ts` 앱 부팅 데이터에 포함된 기본 프롬프트(`defaultPromptKo`)가 사용자 의사와 상관없이 강제 적용되던 문제를 해결하고, 전적으로 사용자의 명시적 입력에 의존하는 시스템으로 전환했습니다.

## 2. 주요 변경 사항

### 2.1 bootstrap.ts 프롬프트 연동 제거 (`App.tsx`)
- **자동 입력 방지**: 템플릿을 처음 로드하거나, 홈에서 진입하거나, 편집 중 템플릿을 변경할 때 `defaultPromptKo`를 `promptKo` 상태에 덮어씌우던 모든 로직을 삭제했습니다.
- **초기 상태 초기화**: 앱 시작 시 프롬프트 입력창이 비어있는 상태(`''`)로 시작하게 하여 사용자가 백지상태에서 자유롭게 배경을 묘사할 수 있도록 했습니다.

### 2.2 사용자 입력 기반 스타일 조합 방식 강화
- 배경 생성 시 4가지 변동성(**고급지게, 빈티지, 세련되게, 카툰화**)이 적용될 때, `bootstrap.ts`의 문구는 더 이상 포함되지 않습니다.
- **현재 적용 로직**: `(사용자 입력 프롬프트) + (스타일 키워드)`
  - 예: 사용자가 "눈 내리는 숲" 입력 시 -> "눈 내리는 숲, 분위기 고급지게", "눈 내리는 숲, 빈티지" 등으로 요청.
  - 사용자가 입력을 하지 않았을 경우 -> "분위기 고급지게", "빈티지" 등 스타일 키워드만으로 생성.

## 3. 기대 효과
- 사용자가 원하지 않는 "제품과 텍스트는 그대로 두고..."와 같은 고정 문구가 생성에 간섭하는 현상을 원천 차단했습니다.
- 사용자는 자신이 원하는 배경 묘사에만 집중할 수 있으며, 시스템은 그 묘사에 AI의 감성 스타일 4종을 입혀 최적의 후보군을 제안합니다.

## 4. 향후 작업 가능 사항
- 사용자가 자주 사용하는 커스텀 프롬프트를 저장하고 불러오는 '프롬프트 즐겨찾기' 기능 추가 검토.
- 입력을 어려워하는 사용자를 위해 '고급진', '귀여운', '미래적인' 등의 키워드를 클릭 한 번으로 조합해주는 '프롬프트 도우미(Tag)' UI 도입 가능.
