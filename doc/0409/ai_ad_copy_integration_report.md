# AI 광고 문구 생성 및 가게 정보 연동 결과 보고서

## 1. 개요
초기 페이지(initPage)에서 입력한 정보를 편집 페이지(editing)로 전달하고, 이를 활용하여 실제 AI 모델 기반의 광고 문구를 생성하는 기능을 구현했습니다.

## 2. 상세 구현 내용

### 2.1 데이터 브리지 구축 (`server/api/storeInfo.js`) [NEW]
- `localStorage`를 활용하여 페이지 간 데이터(가게 이름, 업종, 주소, 편의시설 등)를 유실 없이 전달하는 유틸리티를 추가했습니다.
- `buildAdPrompt()` 메서드를 통해 사용자가 선택한 편의시설 정보만 필터링하여 최적화된 프롬프트를 생성하는 기능을 포함합니다.

### 2.2 정보 저장 로직 추가 (`initPage/App.jsx`)
- 사용자가 원하는 디자인을 클릭(`handleSelectDesign`)하여 편집 페이지로 넘어가기 직전, 현재까지 입력된 모든 가게 정보를 `storeInfo` 유틸리티를 통해 저장합니다.

### 2.3 AI 광고 문구 생성 연동 (`editing/App.tsx`)
- 'AI 문구' 버튼 클릭 시 실행되는 `handleGenerateSlogan` 함수를 전면 수정했습니다.
- **연동 흐름**:
  1. `storeInfo`에서 저장된 정보를 불러옴.
  2. 업종, 주소, 편의시설(주차장, 노키즈존 등)이 포함된 상세 프롬프트 구성.
  3. `adverApi.generateAdCopy`를 통해 모델 서버에 생성 요청.
  4. 수신된 결과를 '최종 광고 문구' 입력칸에 자동 반영.

## 3. 결과 확인 방법
1. **initPage**에서 가게 이름, 주소 및 편의시설(예: 주차장, 배달 가능)을 선택해 주세요.
2. 디자인 하나를 선택하여 **편집 창**으로 이동합니다.
3. 사이드바의 **광고 정보** 섹션에서 'AI 문구' 버튼을 눌러주세요.
4. 터미널 및 콘솔 로그를 통해 상세 프롬프트가 전달되고, AI가 생성한 문구가 입력칸에 채워지는지 확인하실 수 있습니다.

## 4. 참고 사항
- 만약 저장된 정보가 없는 경우, 시스템은 안전하게 기존의 로컬 자동 생성(Fallback)을 수행합니다.
- `extraInfo` 중 사용자가 '노출' 여부를 체크하거나 실제로 보유하고 있는 정보들만 프롬프트에 기입되도록 정교하게 필터링했습니다.
