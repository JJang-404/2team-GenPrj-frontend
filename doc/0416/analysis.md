# 구도 선택(WireframeChoiceCard) 및 메인 프리뷰 분석 보고서 (2026-04-16)

본 문서는 '구도 선택' 카드와 '메인 프리뷰' 간의 텍스트 배치 불일치 및 Type 2의 문구 중복 현상에 대한 분석 결과를 담고 있습니다.

## 1. Type 2 광고 문구 중복 현상 분석

`SingleCompactLayout.jsx` (Type 2 레이아웃) 코드를 분석한 결과, 실제로 동일한 광고 문구(`mainSlogan`)가 두 곳에서 렌더링되고 있음을 확인했습니다.

*   **상단(Header)**: `StoreTitle` 바로 아래에 슬로건이 배치됨 (Line 102)
*   **하단(Footer)**: `absolute bottom-0` 컨테이너 내부에 슬로건이 배치됨 (Line 107)

**결과**: Type 2에서만 문구가 상/하단에 두 줄로 중복 노출되는 원인입니다. 사용자 요청에 따라 하단만 남기고 상단은 제거할 필요가 있습니다.

## 2. 타입별 광고 문구(mainSlogan) 위치 현황

현재 각 레이아웃 컴포넌트(`initPage` 소스 기준)에서의 슬로건 위치는 다음과 같습니다.

| 타입 | 레이아웃 컴포넌트 | 슬로건 위치 | 비고 |
|---|---|---|---|
| **Type 1** | `SingleLargeLayout` | **하단** (bottom-0) | 일치 |
| **Type 2** | `SingleCompactLayout` | **상단 & 하단** | **중복 발생** |
| **Type 3** | `OverlapGroupLayout` | **하단** (bottom-0) | 일치 |
| **Type 4** | `HalfCropGroupLayout` | **하단** (bottom-0) | 일치 |

## 3. 메인 프리뷰 좌표와의 불일치 확인

현재 `wireframeTextPlacements.ts` 설정값은 다음과 같습니다.

*   **Type 1**: `y: 92%` (하단)
*   **Type 2**: `y: 15%` (상단 헤더) -> **컴포넌트의 하단 배치와 불일치**
*   **Type 3**: `y: 92%` (하단)
*   **Type 4**: `y: 93%` (하단)

**분석**: Type 2의 경우 메인 프리뷰와 선택지가 일치하지 않는 근본 원인은 `wireframeTextPlacements.ts`에서 슬로건을 상단(`15%`)으로 설정했으나, 정작 실제 보여지는 레이아웃 컴포넌트에서는 하단에도 문구가 존재(중복)하기 때문입니다.

## 4. 향후 조치 제안 (작업 대기)

1.  **Type 2 코드 수정**: `SingleCompactLayout.jsx`에서 상단 슬로건 렌더링 코드를 제거(혹은 주석 처리).
2.  **좌표 동기화**: `wireframeTextPlacements.ts`에서 Type 2의 `mainSlogan` 좌표를 상단(`15%`)에서 다른 타입들과 마찬가지로 하단(`92%` 부근)으로 재조정.
3.  **텍스트 정렬**: 카드의 줄바꿈 현상을 방지하기 위해 텍스트 박스 너비(`width`)를 충분히 확보.
