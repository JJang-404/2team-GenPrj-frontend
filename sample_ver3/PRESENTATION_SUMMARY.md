# 발표용 요약

## 1. 우리가 만들고 있는 것

이 프로젝트는 단순한 포스터 생성기가 아니라,  
**소상공인 광고 제작 보조 시스템**이다.

사용자가:

- 가게 정보
- 상품 사진
- 홍보 목표
- 원하는 분위기

를 입력하면,

- 광고 문구를 만들고
- 맞는 템플릿을 고르고
- 최종 광고 이미지를 렌더링하고
- 이후 자연어 수정 요청까지 반영하는 구조를 목표로 한다.

즉, 핵심은 **이미지 생성 자체**가 아니라  
**입력 -> 광고 기획 -> 템플릿 선택 -> 결과물 생성 -> 수정**까지 이어지는 파이프라인이다.

---

## 2. 왜 이 문제가 중요한가

소상공인은 보통 아래 문제를 가진다.

- 디자인을 직접 만들기 어렵다
- 광고 문구까지 같이 만들기 어렵다
- 이미지 생성 AI만 써서는 실제 광고 결과물이 바로 나오지 않는다
- 가게 정보와 메뉴 정보가 반영되지 않으면 품질이 낮다

따라서 이 서비스는 단순 이미지 생성이 아니라  
**광고용 결과물을 빠르게 만드는 도구**가 되어야 한다.

---

## 3. 버전 흐름

### ver1

- 규칙 기반 템플릿 선택
- 규칙 기반 수정 처리
- 안정적이고 예측 가능
- 하지만 확장성이 제한적

### ver2

- LLM 기반 템플릿 선택
- LLM 기반 수정 요청 해석
- 템플릿 자동 생성 스크립트 추가
- 더 유연하지만 외부 의존성과 실패 가능성이 큼

### ver3

- ver1의 안정성 + ver2의 확장성 결합
- LLM이 실패해도 규칙 기반 fallback으로 계속 동작
- 검증된 템플릿만 기본 경로에 사용
- 기본 실행은 안정형 경로 우선

한 줄로 말하면:

**ver3는 “LLM을 붙였지만, LLM이 실패해도 시스템이 무너지지 않는 구조”다.**

---

## 4. 현재 가장 중요한 구조

```text
사용자 입력
-> 입력 정규화
-> 광고 기획
-> 템플릿 선택
-> 비주얼 생성 또는 원본 이미지 사용
-> 최종 렌더링
-> 수정 요청 반영
-> 프론트 편집
```

이 구조가 중요한 이유:

- 입력이 많을수록 결과 품질이 올라간다
- 템플릿과 문구가 광고 품질을 결정한다
- 최종 렌더링은 안정적이어야 한다
- 사용자는 마지막에 직접 조정하고 싶어한다

---

## 5. ver3의 핵심 설계 원칙

### 1. LLM은 보조다

LLM이 잘 되면 품질이 올라간다.  
하지만 실패해도 기본 동작은 유지돼야 한다.

### 2. 템플릿은 검증 후 사용한다

자동 생성 템플릿은 많아도,  
검수 전까지는 메인 선택 경로에 올리지 않는다.

### 3. 최종 렌더는 deterministic 해야 한다

최종 결과물은 예측 가능해야 하고,  
검증된 JSON을 기반으로 렌더링해야 한다.

### 4. 입력은 많을수록 좋다

이 서비스는 단순 이미지 생성기가 아니라  
가게 정보 기반 광고 제작 도우미이기 때문에  
입력 정보가 많을수록 품질이 좋아진다.

### 5. 최종 UX는 웹 편집기로 닫힌다

백엔드가 초안을 만들고,  
사용자가 프론트에서 마지막 조정을 하는 구조가 현실적이다.

---

## 6. 왜 ver3 구조가 필요한가

ver1만으로는 너무 제한적이다.  
ver2만으로는 너무 불안정하다.

그래서 ver3는 아래 문제를 해결하기 위해 필요하다.

- LLM 실패 시 전체 흐름이 깨지는 문제
- 자동 생성 템플릿 품질이 들쭉날쭉한 문제
- 무거운 생성 경로를 기본값으로 둘 때 첫 실행이 자주 실패하는 문제

ver3는 이걸 해결하기 위해:

- 룰 기반 fallback 추가
- 템플릿 검수 구조 추가
- 안정형 CLI 기본값 적용
- validation 레이어 추가

를 넣었다.

---

## 7. 앞으로의 제품 방향

이 프로젝트는 앞으로 아래 방향으로 가는 것이 맞다.

### 백엔드

- 입력 정규화 레이어 추가
- 광고 기획 / 컨설팅 레이어 추가
- 템플릿 메타데이터 강화
- 자동 생성 템플릿 canonical 변환

### 프론트엔드

- 웹 편집기 제공
- 텍스트 위치, 크기, 색상 수정
- 배지/가격 강조 수정
- 최종 저장 및 다운로드

### 비즈니스

- 카페 / 디저트 / 음식점 중심 초기 타겟
- 광고 생성 + 수정 + 컨설팅 패키지 가능성
- 업종별 템플릿팩 확장 가능

---

## 8. 발표용 결론

이 프로젝트의 핵심은:

**“사진과 가게 정보를 넣으면 광고 기획부터 결과물 생성, 수정까지 이어지는 소상공인 광고 제작 보조 시스템을 만드는 것”**이다.

그리고 `sample_ver3`는 그 목표를 위해,

- LLM의 유연함은 살리고
- 시스템 안정성은 유지하고
- 실제 서비스 구조로 확장 가능한 형태로

다시 정리한 버전이다.
